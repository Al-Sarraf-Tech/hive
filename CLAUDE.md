# Hive — Project Instructions

## Overview
Hive is a lightweight, cross-platform container orchestrator. Three components:
- **daemon/** — Go daemon (`hived`), manages containers, gossip mesh, gRPC API
- **cli/** — Rust CLI (`hive`), user-facing commands via gRPC to daemon
- **tui/** — Rust TUI (`hivetop`), real-time dashboard via gRPC to daemon
- **proto/** — Protobuf definitions, source of truth for API contracts
- **console/** — Svelte 5 web UI (Phase 3, not yet implemented)
- **recipes/** — TOML-based one-click deploy templates

## Build

```bash
# Go daemon
cd daemon && go build ./cmd/hived

# Rust CLI
cd cli && cargo build

# Rust TUI
cd tui && cargo build

# Everything
make build
```

## Test

```bash
# Go tests
cd daemon && go test ./...

# Rust tests
cd cli && cargo test
cd tui && cargo test

# Everything
make test
```

## Lint

```bash
# Go
cd daemon && go vet ./...

# Rust (both crates)
cd cli && cargo fmt --check && cargo clippy -- -D warnings
cd tui && cargo fmt --check && cargo clippy -- -D warnings

# Everything
make lint
```

## Protobuf

Proto files live in `proto/hive/v1/`. Generated code:
- Go: `daemon/internal/api/gen/hive/v1/` (generated by protoc)
- Rust: built at compile time via `tonic-build` in `build.rs`

To regenerate Go code:
```bash
make proto
```

## Running

```bash
# Start daemon (requires Docker/Podman running)
cd daemon && go run ./cmd/hived --data-dir /tmp/hive-data --log-level debug

# In another terminal, use CLI
cd cli && cargo run -- status
cd cli && cargo run -- deploy ../hive.toml.example

# Or launch TUI
cd tui && cargo run -- --addr 127.0.0.1:7947
```

## Key conventions
- No macOS targets. Linux + Windows only.
- Go daemon uses `go:build` tags for platform-specific code (paths_linux.go, paths_windows.go).
- All containers managed by Hive are labeled `hive.managed=true` and `hive.service=<name>`.
- Hivefile format is TOML (not YAML).
- Secrets use `{{ secret:key }}` placeholder syntax in env vars.
- gRPC is the only communication protocol between components.
